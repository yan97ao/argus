### [diffusion] support tp for ideogram4 (#27393)
**SHA**: `02be2e7` | 🔗 [查看提交](https://github.com/sgl-project/sglang/commit/02be2e71899491b7aaf2849dce6431f61fc190b6)

**🎯 变更类型**：功能增强 / 架构变更 / 性能优化  
**⚡ 重要程度**：🔴高  

**📋 变更摘要**  
1. 为 Ideogram‑4 系列模型新增 Tensor‑Parallel（TP） 支持，实现了列并行的 FP8 权重量化线性层 `WeightOnlyFP8ColumnParallelLinear`，以及对应的列并行 Linear 构造器。  
2. 完善 BitsAndBytes 4‑bit 权重量化的分片逻辑，能够在 TP 场景下只保留本地输出 shard，避免全量加载。  
3. 修复模型加载路径（`fsdp_load.py`）对自定义 Parameter 子类的兼容性，提升 checkpoint 迁移的鲁棒性。  
4. 更新 Vocoder 配置、测试用例以及一致性阈值，以验证 TP+FP8 在多 GPU 环境下的正确性。  

**🎯 影响范围**  
- `sglang/multimodal_gen/runtime/models/dits/ideogram.py`（Ideogram‑4 Diffusion 模型）  
- `sglang/multimodal_gen/runtime/models/encoders/qwen3vl.py`（Qwen‑3‑VL 编码器）  
- `sglang/multimodal_gen/runtime/layers/quantization/weight_only_fp8.py`（FP8 列并行实现）  
- `sglang/multimodal_gen/runtime/layers/quantization/bitsandbytes.py`（4‑bit 分片改造）  
- `sglang/multimodal_gen/runtime/loader/fsdp_load.py`（参数实例化兼容性）  
- 单元/集成测试文件以及 `consistency_threshold.json`  

---

### 🔍 技术洞察  

| 维度 | 影响 |
|------|------|
| **架构影响** | - 引入 **ColumnParallelLinear** 与 **WeightOnlyFP8ColumnParallelLinear**，使得网络的 `Linear` 层在模型并行（TP）下按 **列切分**，并在需要时通过 `tensor_model_parallel_all_gather` 收集输出。<br>- 对 `Qwen3VLTextAttention`、`Qwen3VLTextMLP`、`Ideogram4Transformer2DModel` 等核心模块新增 `tp_size`、`tp_rank` 等属性，实现 **自动分割** 与 **自动聚合**。<br>- `bitsandbytes` 4‑bit 量化的 **sharding** 现在在 `bnb_full_shape`、`bnb_local_shape`、`bnb_output_shard_start` 等元信息的驱动下完成，仅支持列并行（output‑wise）切分。 |
| **性能影响** | - **吞吐量提升**：在多卡（TP=2/4）部署时，每卡只计算本地列，显著降低每卡算子 FLOPs 与显存占用（如 `qkv.weight` 从 `[6912, 4608]` → `[6912, 2304]`）。<br>- **通信开销**：每层线性输出需要一次 `all_gather`（ColumnParallelLinear）或在注意力后聚合，导致额外的 NCCL/torch.distributed 通信，通信量≈输出特征维度/TP。<br>- **加载速度**：`fsdp_load.py` 中对自定义 Parameter 的兼容处理，避免因 `__new__` 参数签名不匹配导致的异常，提升 checkpoint 恢复的稳定性。 |
| **安全考虑** | - 变更均为算子/调度层面，不涉及网络、文件系统权限或外部依赖，未引入新的安全风险。<br>- 唯一需要注意的是 **量化状态（quant_state）** 的切片逻辑必须确保不泄露未对齐的内存块；当前实现加入对对齐和 blocksize 的严格检查，有效防止潜在的内存越界。 |
| **可维护性** | - 新增了大量 *helper* 函数（`_tp_world_size`, `_make_text_linear`, `_gather_tensor_parallel_activation`），代码结构更加模块化，易于在其他模型中复用。<br>- 测试覆盖率显著提升：新增 **TP 场景** 单元测试、`bitsandbytes` 分片单元测试以及一致性阈值的更新，降低回归风险。 |

### ⚠️ 潜在风险  

1. **分片对齐要求**  
   - `bitsandbytes` 4‑bit TP 只能在 **列并行**（output shard）且 **blocksize 对齐** 时使用。若用户自行配置不满足 `output_start % blocksize == 0` 或 `local_numel % blocksize == 0`，将抛出 `ValueError`。  
   - 对于 **非 2 的幂** 的 TP size 或 **不整除** `out_features` 的层，当前实现会回退到普通（非列并行）Linear，可能导致显存不均衡或性能不达预期。

2. **全局跨进程参数共享**  
   - `tensor_model_parallel_all_gather` 依赖 NCCL/torch.distributed 正确初始化。若用户在单进程或未启动 `torchrun --nproc_per_node` 环境下使用 `tp_size>1`，会触发 `group is None` 或 deadlock。  
   - 代码在 `WeightOnlyFP8ColumnParallelLinear` 中默认 `gather_output=True`，但在某些子图（如注意力的 Q、K、V 投影）显式设为 `False`，若误用而忘记聚合，可能导致形状不匹配错误。

3. **checkpoint 兼容性**  
   - 旧版 checkpoint 保存的 `weight` 仍为完整形状；在 TP 环境下加载时会被切分。若 checkpoint 已经是 **列并行** 预切分（例如使用外部工具手动切分），当前加载逻辑会再次切分，导致尺寸错误。  
   - `fsdp_load._make_param_like` 现在尝试 `nn.Parameter` 回退，但仍依赖原参数的 `__dict__` 完整复制；自定义属性（如 `bnb_*`）若在原参数上缺失，需要额外手动设置。

4. **调试复杂度**  
   - 增加的分布式属性（`tp_rank`, `tp_size` 等）使得错误信息更难定位。建议在异常抛出时打印 **TP rank** 与 **分片尺寸**，以帮助用户快速定位问题。

### Fix PP is_fully_idle missing in-flight microbatches (#27446)

**🎯 变更类型**：Bug 修复 / 功能增强  
**⚡ 重要程度**：🟡 中  

**📋 变更摘要**  
1. `Scheduler.is_fully_idle` 在流水并行（PP）场景下未考虑在‑flight 微批导致的假空闲，新增 `_pp_microbatches_drained` 判空逻辑并在 `is_fully_idle` 中使用。  
2. `ScriptedContext` 的请求遍历函数 `_get_all_reqs` 适配 PP，多微批 (`mbs`、`last_mbs`、`running_mbs`) 时统一收集请求。

- `python/sglang/test/scripted_runtime/` 相关脚本及查询函数。  
- `test/registered/chunked_prefill/` 新增多 GPU 测试。  

**💡 关注建议**

- 对 PP=1 的路径保持兼容；建议在 `is_fully_idle` 前加入注释说明新判空的语义。

### Complete server warmup before scripted runtime scripts start (#27445)
**SHA**: `14b8f98` | 🔗 [查看提交](https://github.com/sgl-project/sglang/commit/14b8f98a21aaceec017b458de699bbfad3eb825b)

**变更类型**：功能增强（针对 scripted runtime 的服务器预热逻辑）  
**影响范围**：`python/sglang/test/scripted_runtime/*` 以及 `sglang.srt.managers.io_struct`（间接引用）。  

**核心改动**  
1. **HTTP 就绪检查**：`_await_http_ready` 由原先轮询 `/health`（返回 503 时仍会阻塞）改为轮询 `/model_info`，仅验证 socket 已绑定且路由已注册，避免在脚本驱动的调度器上触发实际生成检查。  
2. **预热驱动**：新增 `_drive_engine_through_warmup`，在调度器启动后持续调度引擎直至预热请求被完整处理并进入两轮空闲状态。超时设为 120 s，失败时抛出明确的 `RuntimeError`。  
3. **工作请求计数**：`ScriptedTokenizerRecvProxy` 增加 `work_reqs_seen`，在接收工作请求时递增，用于判断预热期间是否真的有推理流量。  
4. **常量 & 参数**：`WARMUP_DRIVE_TIMEOUT_S`、`LISTENER_ACCEPT_TIMEOUT_S`（注释说明）以及对 `skip_server_warmup` 标志的兼容处理。  

**潜在影响**  
- 预热阶段会额外消耗 CPU/GPU 资源，测试时间可能延长。  
- `idle_streak` 计数依赖 `pp_size` 与 `pp_async_batch_depth`，若配置非常大或非标准微批次，可能导致不必要的等待或误判。  
- 新增的 `io_struct` 导入在运行时几乎无负担，但若未来迁移或裁剪模块，需确认仍可访问。  

**建议**  
- 在 CI 中加入对 `skip_server_warmup` 为 `True` 的快速路径验证，防止因预热超时导致整套测试卡死。  
- 为 `_drive_engine_through_warmup` 编写单元测试，覆盖超时、正常完成以及 `work_reqs_seen==0` 的情况。  
- 在文档或注释中明确说明 `WARMUP_DRIVE_TIMEOUT_S` 与 `LISTENER_ACCEPT_TIMEOUT_S` 的关系，方便后续调参。  
- 考虑在 `idle_streak` 检查中加入上限阈值，以免在异常长的 PP 配置下出现无限循环。  

总体而言，此次改动提升了 scripted runtime 在预热阶段的可靠性，避免了原先因 `/health` 检查阻塞导致的脚本卡死，但需要关注超时阈值与空闲检测逻辑的稳健性。

### [spec] Dedup draft `kv_indices` sizing into `spec_utils` helpers (#27475)
**SHA**: `eab2e02` | 🔗 [查看提交](https://github.com/sgl-project/sglang/commit/eab2e02fa0c49595098a715d331cb9ba10692c63)

**🎯 变更类型**：功能增强 / Bug 修复（统一 `kv_indices` 尺寸计算）  
**⚡ 重要程度**：🟡 中  

**📋 变更摘要**  
1. 在 `spec_utils.py` 中新增 `draft_kv_indices_buffer_width` 与 `draft_kv_indices_used_len` 两个工具函数，统一计算 Eagle‑draft‑decode 场景下 `kv_indices` 缓冲区的行宽和已使用长度。  
2. 四个注意力后端（aiter、flashinfer、flashinfer_mla、triton）均改为调用上述 helpers，去除硬编码 `batch_size * topk * max_context_len` 等公式，并在 `common_template` 中使用 `draft_kv_indices_used_len` 做容量断言。  

**🎯 影响范围**  
- `python/sglang/srt/layers/attention/*_backend.py`（四个后端实现）  
- `python/sglang/srt/speculative/spec_utils.py`（新增函数）  

**💡 关注建议**  
1. **正确性验证**：确保 `draft_kv_indices_buffer_width` 在所有后端的 `dtype`（int32/int64）和设备（CUDA/MUSA）上返回的值一致；建议加入单元测试，对极端参数（`batch_size=0`、`topk=1`、`max_context_len=1`）进行断言。  
2. **向后兼容**：虽然新函数只做等价计算，但若历史代码仍硬编码该公式，需搜索项目确认已全部迁移，防止遗漏导致潜在 OOB。  
3. **性能检查**：函数本身为纯算数，开销可忽略；但请在性能基准上确认引入的额外 Python 调用没有影响高频路径（如每个 token 推理的 `common_template`）。  
4. **文档与注释**：`spec_utils.py` 已加入 docstring，建议在各后端文件相应位置添加简要说明，提醒后续维护者使用这些 helpers 而非手写公式。  
5. **异常信息**：现在的断言信息已指明需要的长度，保持即可；若项目使用自定义异常机制，可考虑统一包装为 `RuntimeError`，便于上层捕获。
