### [NPU] Add Gemma4 Sliding Window Attention support on Ascend backend (#26147)
**SHA**: `3a3a759` | 🔗 [查看提交](https://github.com/sgl-project/sglang/commit/3a3a759464041f83ad4fa3d7751117b45d0629a2)

**🎯 变更类型**：功能增强  
**⚡ 重要程度**：🔴高  
**📋 变更摘要**：在 Ascend NPU 后端为 Gemma‑4 系列模型引入 Sliding‑Window Attention（SWA）支持，新增 SWA‑mask、索引缓冲、Hybrid‑SWA 判定以及 TND 布局优化。同步更新 Torch‑Native NPU 后端以兼容滑动窗口、改进跨模态预处理、拓宽后端白名单，并在测试套件中加入 Gemma‑4 系列模型的 GSM8K 与 MMMU 基准验证。  

**🎯 影响范围**：  
- `sglang/srt/hardware_backend/npu/attention/ascend_backend.py`（核心注意力实现、CUDA‑Graph 元数据、SWA mask 生成）  
- `sglang/srt/hardware_backend/npu/attention/ascend_torch_native_backend.py`（SDPA 前向扩展/解码路径）  
- `sglang/srt/models/gemma4_causal.py` & `gemma4_mm.py`（模型配置、跨模态 token 处理）  
- `sglang/srt/server_args.py`（后端白名单加入 `ascend`）  
- 测试目录 `test/manual/ascend/*`（新增 Gemma‑4 系列模型的 GSM8K 与 MMMU 基准）  

---

### 🔍 技术洞察

| 维度 | 影响描述 |
|------|----------|
| **架构影响** | 1. 在 `AscendBackend` 中加入 **Hybrid‑SWA 判定**（`_is_swa_layer`），并在 `ForwardMetadata` 中新增 `swa_mask` 用于图模式下的掩码。<br>2. 为图捕获预分配 **mask** 与 **indices** 张量，避免在每次 replay 时重新创建，提高图的可复用性。<br>3. 引入 `full_to_swa_index_mapping`，在多池（Full ↔︎ SWA）场景下实现 KV 索引映射，保持统一的 KV 接口。<br>4. 在 Torch‑Native 后端中加入 `sliding_window_size`、`full_to_swa_mapping` 参数，统一前向/解码路径的滑动窗口逻辑。 |
| **性能影响** | - **正向**：对 Gemma‑4‑26B、31B 等大模型使用 Sliding‑Window 可以显著削减 KV 长度 (O(window) → O(seq))，降低 NPU memory 带宽与算子调度开销。<br>- **负向**：在图模式下每次 replay 需要计算 `mask = (indices < start) | (indices >= end)`，该算子为一次 `torch` 布尔比较，开销在 1‑2 ms 量级（相较于原始全注意力可接受）。<br>- **内存**：额外分配 `swa_mask` (bool) 与 `swa_indices` (int32) 与 batch大小线性增长，最大约 1 MiB/bs（对 2‑4 bs 影响可忽略）。<br>- **算子选择**：对 `qk_head_dim == v_head_dim` 且为 128/192 的情况使用 TND 布局，提升 NPU 端算子并行度；否则回退 BSND，保持兼容性。 |
| **安全考虑** | - 掩码生成逻辑直接影响信息泄露风险：若 `swa_mask` 错误地留下 `True`（mask）位置为 `False`（未掩），会导致后续 token 访问超窗口信息。代码已在 `forward_decode_graph` 中对非 SWA 层使用 `None`，并在图捕获阶段对未使用的 batch 填充 `True` 以确保安全默认。<br>- 新增的 `full_to_swa_index_mapping` 只在内部使用，无外部输入，未产生注入面。 |
| **可维护性** | - 将判断逻辑抽象为 `_is_swa_layer` 与 `_can_use_tnd`，降低后续修改耦合度。<br>- `ForwardMetadata` 增加字段 `swa_mask`，但未修改已有序列化/日志路径，保持向后兼容。<br>- 对跨模态 prefill 中的 `ple_ids` 替换为 `torch.where`，兼容 NPU 图模式的稀疏写入。 |

### ⚠️ 潜在风险

1. **掩码边界错误**  
   - `starts = torch.clamp(seq_lens_int - self.sliding_window_size, min=0)` 与 `mask` 生成的逻辑若 `sliding_window_size` 为负或异常大，可能产生全 `True`（全遮蔽）或全 `False`（全未遮蔽）情况，导致结果不正确或显存暴增。  
2. **映射错误**  
   - `full_to_swa_index_mapping` 未在所有 KV‑pool 路径统一检查，若某层未设置但仍传入映射，会触发 `NoneType` 错误。  
3. **图捕获兼容性**  
   - 在 CUDA‑Graph（Ascend Graph）捕获期间，`seq_lens_cpu` 仍可能为 `None`（如首次捕获），导致 `metadata.swa_mask` 填充全 `True`，后续 replay 时若未更新会影响推理准确性。  
4. **内存回撤**  
   - 对极大 batch（>8）和极长序列（>16k）时，`swa_mask` 与 `swa_indices` 预分配可能导致显存压力，尤其在多卡 `tp_size=2` 场景。  
5. **兼容性回归**  
   - 旧模型（不使用 sliding‑window）在 `use_sliding_window_kv_pool` 为 `True` 时仍会走 SWA 代码路径，需要确保 `layer.sliding_window_size==-1` 正确跳过 mask 生成。

### 💡 关注建议

1. **单元/集成测试**  
   - 为 `swa_mask` 生成逻辑加入边界值测试（window=0、window=full、seq_len < window）。  
   - 验证 `full_to_swa_index_mapping` 在 `Hybrid‑SWA` 与 `Pure‑SWA` 两种 KV‑pool 配置下的正确性。  

2. **性能基准**  
   - 在不同 `sliding_window_size`（64、128、256）与 batchsize 下跑 `test_npu_gemma_*`，记录显存、吞吐和延迟，确保改动带来预期的 30%‑50% 加速。  

3. **监控安全掩码**  
   - 在推理结束后可加入断言：`assert not (output_masked & ~expected_mask).any()`，捕获意外未掩蔽的 token。  

4. **文档/示例更新**  
   - 在 `README` 与 `sglang/srt/models/gemma4_*.py` 中明确 `sliding_window` 参数含义（-1 表示关闭），并提供 `--attention-backend ascend` 示例。  

5. **资源预估**  
   - 在 `server_args` 中对 `--mem-fraction-static` 与新的 mask/indices 大小进行提醒，防止用户在显存紧张的机器上出现 OOM。  

6. **回滚路径**  
   - 保持 `use_sliding_window_kv_pool` 为 `False` 时的旧代码路径不受影响，以便在 Ascend 环境出现异常时快速回退。

### [Spec] Remove the dead spec V1 scheduler paths (#27977)
**SHA**: `3ffe725` | 🔗 [查看提交](https://github.com/sgl-project/sglang/commit/3ffe72517f8d6c0f45630bc5ffa04ec37590bb57)

**🎯 变更类型**：重构 / 架构变更  
**⚡ 重要程度**：🔴 中  
**📋 变更摘要**：本次提交删除了已废弃的 Spec V1（旧的草稿扩展/验证）调度路径及其相关实现，统一所有推理逻辑为 Spec V2（Eagle 系列）或非推测模式。主要措施包括：  
1. 代码层面去除 `is_spec_v2`、`supports_spec_v2` 等旧属性，改为统一使用 `spec_algorithm.is_none()` 判断是否开启推测。

3. 调整 KV‑pool 取稿逻辑、内存分配策略以及测试用例，以兼容仅保留的 Spec V2 调度。  

**🎯 影响范围**：  
- `sglang/srt/layers/attention/dsa_backend.py`（注释更新）  
- `sglang/srt/layers/utils/logprob.py`（删除大量 Spec V1 相关逻辑）  
- `sglang/srt/managers/*`（`overlap_utils.py`、`schedule_batch.py`、`scheduler.py`）  
- `sglang/srt/managers/scheduler_components/batch_result_processor.py`（V1 输出处理移除）  
- `sglang/srt/mem_cache/*`（KV‑pool 获取、内存分配统一）  
- `sglang/srt/speculative/eagle_info.py`（完整删除 Spec V1 `verify` 实现）  
- `sglang/srt/speculative/spec_info.py`、`spec_registry.py`（API 调整、弃用警告）  
- 相关单元测试与基准测试文件的更新

6. **回滚兼容**：若有旧模型依赖 `cat` 方式的副作用（如显式保持原始 `embeds` 的拷贝），请评估是否需要保留兼容层或在文档中注明行为已改为“原位替换”。  

总体来看，此修复消除了跨序列嵌入错位的根本原因，提升了多模态批量推理的准确性；在正式发布前完成上述验证即可。
