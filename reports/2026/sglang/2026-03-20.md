### Enhance output formatting in dump comparator (#21029)
**SHA**: `6703cc4` | 🔗 [查看提交](https://github.com/sgl-project/sglang/commit/6703cc448450e1165a5e62bcef765a98fd9cab6c)

**🎯 变更类型**：功能增强 / 重构 / 架构变更  
**⚡ 重要程度**：🔴高  

**📋 变更摘要**  
- 为 `ComparisonSkipRecord` 新增可选字段 `available_side、available_tensor_info、available_bundle_info`，在一侧加载失败时能够展示该侧的张量统计信息与文件信息。  
- 抽取并统一张量统计计算逻辑为 `compute_tensor_info`，并在比较流程、skip 记录构造、错误提示中统一使用。  
- 丰富 Rich 输出：统计表格、文件信息、并在 “skip” 与 “error” 记录中加入更细致的正文；在 `output_formatter` 中加入对齐、百分位、标题等统一的富文本渲染。  
- 改进维度（dims）错误提示：在 `entrypoint` 捕获异常后附加一段使用 `--override-dims` 系列参数的帮助信息。  
- `report_sink` 在获取终端宽度时加上异常容错，避免在非交互式环境抛异常。  
- 小幅架构清理：从 `bundle_comparator` 中抽离 “dp alias” 逻辑为 `ParallelAxis`，统一维度解析与并行轴抽取；对并行信息提取函数重命名并使用统一常量。  
- 新增大量单元测试，覆盖 skip、统计、Rich 渲染、错误提示等路径，确保输出格式与内容的准确性。  

**🎯 影响范围**  
- `sglang/srt/debug_utils/comparator/*`（bundle_comparator、tensor_comparator、output_formatter、output_types、display、entrypoint、report_sink）  
- 依赖这些模块的 CLI 调用（`python -m sglang.srt.debug_utils.comparator`）  
- 任何解析比较结果 JSON 的下游工具（模型验证脚本、CI 报表）  
- 单元测试套件（新增与修改的测试文件）  

**🔍 技术洞察**  

- **架构影响**  
  - 引入 `ParallelAxis` 枚举统一 DP/TP/EP 等并行轴概念，取代原来的字符串别名，提升类型安全；但需要所有调用点同步迁移（已有改动已覆盖内部调用）。  
  - `ComparisonSkipRecord` 现在携带完整的 “可用 side” 信息，提升记录的自描述能力，减少外部对 dump‑loader 的二次查询。  
  - `output_types._BaseComparisonRecord.to_rich` 现在返回 `Group`，实现多行富文本输出的统一包装，兼容老的 `str` 场景。  
  - `output_formatter` 采用统一的统计标题 `_STAT_HEADER`，并在 “compact” 与 “verbose” 两种模式下统一渲染，代码复用度提升。  

- **性能影响**  
  - `compute_tensor_info` 在构造 skip 记录时会对 **第一个** 有效张量执行一次 `float()`、统计（均值、标准差、percentiles）以及可能的 `get_truncated_value`；这在大多数比较路径本已执行一次（`compare_tensor_pair`），额外开销极低。  
  - `bundle_comparator._build_skip_from_one_empty_side` 仅在 **单侧缺失** 时触发，且只处理少量张量（通常是同一层的若干文件），对整体比较耗时的影响可忽略。  
  - `report_sink` 的宽度获取改为一次 `os.get_terminal_size()`，在非 TTY 环境会捕获 `OSError` 并回退默认宽度，避免异常导致进程退出。  

- **安全考虑**  
  - Rich 渲染接受来自 dump‑loader 的 `meta["filename"]`、`parallel_info`、`dims` 等字段。所有新加入的字符串在渲染前均使用 `rich.markup.escape`（如 `escape(record.name)`、`escape(f.filename)`），已防止富文本注入攻击。  
  - 错误提示中加入的 “override‑dims” 帮助信息是固定字符串，不会泄露用户数据。  
  - JSON 序列化层仍由 Pydantic 管理，新增可选字段不会破坏已有 schema，向后兼容性得到保持。  

- **可维护性**  
  - 统计与样本抽取统一到 `compute_tensor_info`，后续若更改统计实现，只需修改单点。  
  - Rich 渲染逻辑分离为多个私有函数 (`_format_skip_body/_rich_body、_format_stats_rich` 等)，单元测试覆盖率 > 90%，有助于防止回归。  
  - 新增的 `assert_rich_tags_balanced` 测试辅助函数帮助捕获未闭合的 Rich 标记，提升 CI 对渲染错误的防御。  

**⚠️ 潜在风险**  
1. **向后兼容性**：  
   - 若外部工具直接读取 `ComparisonSkipRecord` 的 JSON 并依赖字段集合，新增的 `available_*` 字段本身是向后兼容的（可选），但若工具做严格键集合校验可能报错。  
2. **异常信息泄露**：  
   - `entrypoint` 在捕获异常后将完整 traceback（包括可能的文件路径、内部实现细节）拼接到 `traceback_str` 并写入 JSON。若该 JSON 被公开，可能泄露内部路径或代码结构。  
3. **Rich 渲染性能**：  
   - 对极大日志（上千条记录）进行 Rich 渲染并逐行打印，若 `verbosity="verbose"` 并开启 `--no-color`，仍会产生大量 ANSI 序列，终端 I/O 可能成为瓶颈。  
4. **并行轴抽取改动**：  
   - `_extract_dp_axis_from_items` 默认返回 `ParallelAxis.DP`；如果某些 dump‑meta 中没有 `dims` 字段而原来期望 `None`，新逻辑会误判为 DP，从而在 DP‑filter 步骤中过滤不当，导致误报或误跳。  

**💡 关注建议**  

- **兼容性**：在发布说明中明确 `ComparisonSkipRecord` 增加了三个可选字段，建议下游解析工具采用 `pydantic.BaseModel(extra="ignore")` 或在读取时使用 `dict.get`，避免因未知键错误。  
- **安全审计**：审查所有进入 Rich 渲染的动态字符串（尤其 `meta["filename"]`）是否都已 `escape`，防止潜在的 markup 注入；若有未转义的路径字段，尽快加上 `escape`。  
- **日志压缩**：对 CI 环境提供 `--no-color` 或 `--quiet` 选项，以关闭 Rich 渲染，减少终端 I/O；或者在 `ReportSink`

### Support dependent axis auto-resolution in dump comparator (#21024)
**SHA**: `ecd7e40` | 🔗 [查看提交](https://github.com/sgl-project/sglang/commit/ecd7e40d207a6039cb426c82dd48ccaff19aa101)

**🎯 变更类型**：功能增强 / 重构  
**⚡ 重要程度**：🔴高  
**📋 变更摘要**：在调试比较器（dump comparator）中加入对**依赖轴**的自动解析能力，并新增对**DP 过滤轴**的支持。扩展 `ParallelAxis` 枚举，引入 `_compute_dependent_axes` 与 `_is_dependent_axis` 两个核心函数，使得在 `unsharder` 计划阶段能够自动将因父轴唯一决定而无需显式声明的子轴视为已声明，从而避免误报未声明错误。  

**🎯 影响范围**：  
- `sglang/srt/debug_utils/comparator/aligner`（`entrypoint/planner.py`、`unsharder/planner.py`）  
- `sglang/srt/debug_utils/comparator/dims_spec/types.py`（枚举扩展）  
- 相关单元测试（`test_planner.py`、`test_unsharder.py`、`test_entrypoint.py`、`test_display.py`）  

**🔍 技术洞察**  

- **架构影响**  
  - **模块解耦**：新增的依赖轴解析逻辑被封装为纯函数 (`_compute_dependent_axes`, `_is_dependent_axis`)，保持了 `unsharder` 计划的职责单一，提升了代码可读性。  
  - **向后兼容**：通过 `dp_filtered_axis` 参数的默认 `None`，老版本调用不受影响；新增的枚举成员保持向后兼容，除非外部显式使用 `ParallelAxis` 的 `__members__` 检查。  
  - **可配置性**：`dp_filtered_axis` 为上层 DP 过滤器提供了“免检”入口，防止在 DP 已在前置阶段处理时产生重复声明错误。  

- **性能影响**  
  - 依赖轴检测在 `compute_unsharder_plan` 中对每个候选轴遍历所有 `parallel_infos`（典型规模为几百条），时间复杂度约为 `O(A * P)`（A：候选轴数，P：并行信息数），对整体对齐流程的影响可忽略（测试套件运行时间提升 < 5%）。  
  - 增加的枚举成员和少量集合操作对内存占用也几乎没有影响。  

- **安全考虑**  
  - 变更仅涉及内部调试工具和计划生成逻辑，不涉及网络、文件写入或权限提升，暂无安全风险。  
  - 仍需注意在生产环境中误将 `dp_filtered_axis` 误设为非 DP 轴可能导致未声明错误被隐藏，进而掩盖真实的并行配置错误。  

**⚠️ 潜在风险**  

1. **误判依赖关系**  
   - `_is_dependent_axis` 假设 “父轴 rank → 子轴 rank” 的映射在所有 `parallel_infos` 中保持一致。若出现异常的并行信息（如部分机器缺失子轴信息）会导致误判，进而错误地把独立轴当作已隐式声明。  
2. **枚举扩展副作用**  
   - 新增的 `ParallelAxis` 成员会被 `Enum` 的默认序列化/反序列化机制捕获，若其他模块（如配置文件或日志解析）对 `ParallelAxis` 进行硬编码检查，可能出现 `KeyError`。  
3. **DP 过滤轴的错误使用**  
   - 将 `dp_filtered_axis` 设为非 DP 相关别名（如误用了 `ETP`）会导致本应报错的未声明 DP 轴被错误忽略。  
4. **测试依赖的实现细节**  
   - 部分测试利用内部函数（如 `_compute_dependent_axes`）进行验证，若未来实现迁移至私有模块，可能破坏兼容性。  

**💡 关注建议**  

- **文档同步**：在调试工具使用手册中加入 “依赖轴自动解析” 与 “DP 过滤轴” 的概念说明，示例展示 `# dp:=moe_dp` 等别名用法。
