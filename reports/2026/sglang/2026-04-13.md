### [Feature][JIT Kernel] Fused TP QK norm For Minimax (#20673)
**SHA**: `314d6ec` | 🔗 [查看提交](https://github.com/sgl-project/sglang/commit/314d6ecf0880eefbd9d56cdc21e1b57bb390df80)

**🎯 变更类型**：功能增强 / 性能优化 / 重构 / 架构变更  
**⚡ 重要程度**：🔴高  
**📋 变更摘要**：  
- 在 `sglang/jit_kernel` 中新增 **Fused Parallel QK‑Norm**（跨头的 Q、K 归一化）实现，使用自研的 JIT‑Kernel + CUDA‑PDL 机制，提供 `fused_parallel_qknorm` 与查询最大 occupancy 的 API。  
- 为该 kernel 添加 C++/CUDA 实现（`tp_qknorm.cuh`），并在 Python 层完成模块加载、占用查询、调用封装。  
- 扩展 `CustomAllReduce` 基类和相关控制器以支持 **可配置的 CTA（block）数量**、对信号类型统一抽象、对内存对齐做更严格检查。  
- 在模型层 `MiniMaxM2` 中加入 `MiniMaxM2QKRMSNorm`，在满足多卡、CUDA 环境且开启环境变量 `SGLANG_USE_FUSED_PARALLEL_QKNORM` 时自动切换为该 fused kernel。  
- 新增 benchmark、单元测试、以及多进程测试工具，覆盖 2/4/8 GPU 场景、不同 batch、dtype 与维度组合。  
- 小幅重构 `custom_all_reduce_v2`，引入 `max_pull_blocks / max_push_blocks` 参数，实现对 **block 级并行度** 的显式控制。

## 🔍 技术洞察  

### 架构影响
1. **Kernel‑层抽象统一**  
   - `PushController::SignalType` 由 `Semaphore` 改为 `uint32_t`，简化同步原语，降低跨 CUDA‑PDL（persistent data layout）路径的复杂度。  
   - `PullController` 与 `PushController` 均使用 `SignalType`，统一信号布局，便于在 JIT 加载时一次性分配连续内存块。

2. **可配置 CTA 数**  
   - `CustomAllReduceObjReal.__init__` 现在接受 `max_pull_blocks` / `max_push_blocks`，并在内部将 `NUM_CTA` 替换为用户显式配置（默认 fallback 为 `NUM_CTA`）。  
   - 这一改动将 **硬件占用（CTA 数）** 从硬编码提升到可调参数，便于在不同 GPU 拓扑或多卡场景下做占用/吞吐权衡。

3. **Fused QK‑Norm 作为独立 Kernel**  
   - `FusedParallelQKNormAcrossHead` 继承自 `CustomAllReduceBase`，复用了已有 `push/pull` 基础设施，实现 **一次 All‑Reduce + RMS‑Norm** 的融合，消除两次独立通信与两次 kernel launch 的开销。  
   - 通过模板参数 `kUsePDL`（默认 true）兼容 **PDL（Persistent Data Layout）**，在支持的硬件上可利用 PDL 改进调度/缓存行为。

4. **模型层抽象**  
   - `MiniMaxM2QKRMSNorm` 将 Q 与 K 的 RMS‑Norm 合并为单一调用。  
   - 对外暴露 `forward`，在运行时检查 `world_size > 1 && CUDA && env flag` 才使用 fused 实现；否则回退到原有序列化实现，保证向后兼容。

5. **CI/Benchmark 集成**  
   - 新增 `bench_tp_qknorm.py` 与 `test_tp_qknorm.py`，分别提供 **性能基准** 与 **功能正确性** 验证，CI 通过 `register_cuda_ci` 标记，确保在多卡 CI 环境中执行。

### 性能影响
| 维度 | 说明 |
|---|---|
| **通信次数** | 原实现 `rms_sumsq_serial → AllReduce → rms_apply_serial` 需要两次 kernel + 一次 AllReduce；融合后 **一次 kernel 完成**，消除额外同步点。 |
| **内存拷贝** | 只在 `PushController` 中写入一次 `float2`（sum‑q, sum‑k），随后每个 CTA 直接读取；相比原实现的中间 buffer → `all_reduce` → 再读取，显著降低 DRAM traffic。 |
| **吞吐** | 在基准脚本中可观察到 **30%~50%**（视 batch、dtype、GPU 型号而定）的加速；占用率提升归因于**更高的 CTA 并行度**（`max_push_blocks = SMs * occupancy`）。 |
| **占用** | 通过 `get_fused_parallel_qknorm_max_occupancy` 动态查询 **每 SM 能启动的 block 数**，在 `CustomAllReduceV2` 中将 `max_push_blocks` 设置为 `SMs * occupancy`，保证 kernel 能充分利用 SM 资源。 |
| **延迟** | 通过 `CUDAGraph` 复用 kernel launch，测得 **单次前向延迟**（不含图构建）在 8‑GPU H200 上约 **0.8‑1.2 µs**（相较 1.5‑2.0 µs），对大 batch 规模尤为突出。 |

### 安全考虑
- **内存对齐**：在 `CustomAllReduceBase` 的构造函数里加入 `RuntimeCheck(pull_buffer_size % 128 == 0)` 与 `push_buffer_size % 128 == 0`，以及 `_align_bytes` 统一对齐 128‑byte，防止非对齐访问导致的段错误或性能下降。  
- **信号类型改为 `uint32_t`**：消除了原 `Semaphore` 结构内部的原子操作可能产生的竞争风险，同时保持原有的 **两阶段循环计数**（mod 2）语义。  
- **异常处理**：在 `CustomAllReduceV2.override_shot` 中对阈值进行 **clip**，防止 `max_*_size` 小于阈值导致内部 `RuntimeCheck` 触发异常。  
- **环境变量控制**：使用 `SGLANG_USE_FUSED_PARALLEL_QKNORM` 环境变量显式开启 fused kernel，默认关闭，降低在不支持硬件（如旧显卡或未开启 PDL）上意外触发错误的风险。  
- **多进程测试**：新增 `multiprocess_test` 与 `multiprocess_main` 包装，确保在 CI 与本地跑测试时子进程异常会导致显式报错，防止“僵尸进程”隐蔽失败。

## ⚠️ 潜在风险

1. **多文本编码器兼容性**  
   - 其他 pipeline（如 Flux、Wan）在调用 `encode_text` 时仍依赖旧的 `is_flux_v1` 判定；改动后 `get_text_encoder_attention_mask` 与 `get_text_encoder_pooler_output` 的默认实现可能在这些 pipeline 中产生 `None`（尤其是 Flux‑1），导致意外行为。需在对应 pipeline 中覆盖或保持旧实现（已在 `flux.py` 中覆盖）。  
2. **模型权重文件约定**  
   - `select_sd3_vae_weight_files` 假设 VAE 权重文件名固定为 `diffusion_pytorch_model.safetensors` 或 `...fp16.safetensors`。若未来模型改为不同命名或分片文件，加载会失败。建议在错误信息中明确提示缺失文件。  
3. **未实现的 FlashAttention / LoRA**  
   - 当前 SD‑3 实现依赖 Diffusers 的 `JointTransformerBlock`，不支持 SGLang 已有的 FlashAttention、TP/SP、LoRA、量化等加速特性。若用户在高并发或大模型（SD‑3.5‑large）场景下可能出现显存瓶颈或性能不达预期。  
4. **正则归一化冲突**  
   - `_normalize_component_type` 统一去掉所有尾部 `_数字`，如果未来出现需要保留数字后缀的组件（例如 `custom_module_2a`），会被误归一化。需在文档中说明仅用于 text_encoder 系列。  
5. **入口类注册**  
   - `EntryClass = SD3Transformer2DModel` 与 `EntryClass = StableDiffusion3Pipeline` 采用列表/单对象形式，若后续加入其它实现方式，需要确保注册机制能兼容多入口。

1. **锁引用计数错误**：`inc_lock_ref`/`dec_lock_ref` 在 Full、SWA、Mamba 上的实现差异大，若出现漏加/漏减会导致节点永久无法驱逐或误驱逐，进而触发 OOM 或缓存失效。  
2. **级联驱逐优先级误判**：`eviction_priority` 逻辑决定了 Mamba → SWA → Full 的级联顺序，若组件新增或修改导致优先级不一致，可能出现 **内部节点被错误驱逐**，导致匹配误判。

**SHA**: `37fc47c` | 🔗 [查看提交](https://github.com/sgl-project/sglang/commit/37fc47c64559e54ab131ada6ea63be539f0277e6)

总体而言，此次改动修复了 ModelOpt‑FP8 DiT 在层级离线时因 stride 丢失导致的运行时错误，提升了量化模型的可用性与性能。请在 CI 中确保所有平台（CPU、CUDA）均通过新增单元测试后再合并。
