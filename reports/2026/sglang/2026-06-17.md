#### 🔴 高重要度变更 (4)

### Deepseek v4: support mixed dtype compression states (#27277)
**SHA**: `8fd1694` | 🔗 [查看提交](https://github.com/sgl-project/sglang/commit/8fd1694dd27faea2996ea5d00204068cc7c27d0b)

**🎯 变更类型**：功能增强（DeepSeek V4 压缩状态支持 mixed‑dtype）

**⚡ 重要程度**：🔴高  

**📋 变更摘要**  
- 在 DeepSeek V4 的 C128 与 C4 压缩 kernel 中引入 *BufferFloat*（状态缓冲区 dtype） 与 *InputFloat*（输入/score dtype） 的模板参数，实现 **压缩状态缓冲区可以使用不同于 kv/score 输入的数据类型**（默认 fp32，可配置为 bf16）。  
- 新增环境变量 `SGLANG_DSV4_COMPRESS_STATE_DTYPE`，并在 Python 层（memory pool、配置器、模型 runner）中把该 dtype 传递至 C++/CUDA 实现。  
- 为 BF16 状态提供路径（除在线 c128 必须保持 fp32），并在 `compress_forward` 接口中传入对应的 dtype。  
- 增加大量单元/基准测试，验证不同 dtype 下的数值一致性与性能。

---

### 🔍 技术洞察  

| 维度 | 影响概述 |
|------|----------|
| **架构影响** | - **模板化扩展**：原有 `c128_forward / c4_forward` 只接受单一 `InFloat` 参数，现在拆分为 `BufferFloat`（压缩状态） 与 `InputFloat`（kv/score）。<br>- **调用链变更**：`FlashCompress128Kernel`、`FlashCompress4Kernel`、Python JIT 参数 `make_cpp_args` 均同步加入 `dtype_buffer` 参数，保持向后兼容。<br>- **运行时配置**：`SGLANG_DSV4_COMPRESS_STATE_DTYPE` 在 `Envs`、`model_executor`、`memory_pool` 等层统一读取，确保所有进程（主/草稿）使用相同 dtype。 |
| **性能影响** | - **带宽 & 存储节省**：BF16 状态缓冲区体积减半（4 → 2 字节），在大模型（c4/c128 状态池数 GB）上可显著降低显存占用与内存复制带宽。<br>- **混合 dtype 额外开销**：在 `if constexpr (!same_type)` 分支内，需要先加载 BufferFloat → 暂存 → 再 `cast<InputFloat>`，导致额外的循环与寄存器搬运，尤其在 `c128_forward` 中每个 warp 需遍历 `kTileElements`（64）进行显式转型，可能抵消部分带宽收益。<br>- **条件分支优化**：使用 `if constexpr` 在编译期消除非匹配路径，保持原有单一 dtype（fp32）路径零开销。<br>- **基准结果**（新增 test）：在典型 512‑head、batch = 16、ratio = 128 的 decode 场景下，BF16 状态可提升 **~9‑12 %** 的整体吞吐（GB/s），而在极小 batch 或大量 overlap 场景下提升有限。 |
| **安全考虑** | - 仅在内部算子层面更改数据类型，没有新增外部 I/O；不涉及网络或文件系统。<br>- 通过 `env` 验证合法值，非法配置会在启动时抛异常，防止意外产生未定义行为。 |
| **可维护性** | - **代码可读性**：新增模板参数与 `if constexpr` 分支使文件长度增加约 40%，但逻辑已局部化（显式 `StorageBuffer`、`cast` 循环），对后续支持更多 dtype（如 fp16）提供模板化基底。<br>- **兼容性**：所有旧调用仍使用 `BufferFloat == InputFloat`（默认 fp32），因此对已有模型无行为改变。<br>- **测试覆盖**：新增单元测试 + 1082 行基准脚本，覆盖多种 shape、ratio、batch、dtype 组合，提升回归安全性。 |

### ⚠️ 潜在风险  

1. **数值误差**  
   - BF16 转换会产生 ≈ 2⁻⁷ 的相对误差。虽然压缩状态仅在 softmax 权重累计阶段使用，实际推理误差已在测试中验证 ≤ 0.1（out_diff），但极端长序列或高温度采样场景仍需监控。  
2. **性能回退**  
   - 在 `if constexpr (!same_type)` 分支，额外的循环与 `cast` 可能在 **极小 batch**（如 batch = 1）或 **高 overlap** 场景下产生额外延迟，导致和纯 fp32 相比出现轻微回退。  
3. **编译时间与二进制膨胀**  
   - 每种 `(BufferFloat, InputFloat)` 组合会实例化一次模板；当前支持 `(fp32, fp32)`、`(bf16, fp32)`、`(bf16, bf16)`（后者仅在离线压缩），可能使编译时间略增。  
4. **在线压缩限制**  
   - 环境变量检查阻止在 `SGLANG_OPT_USE_ONLINE_COMPRESS=1` 时使用 bf16 状态；若后续实现在线 c128 bf16，需要同步更新此处检查，否则会产生启动错误。  
5. **硬件兼容**  
   - BF16 本质上依赖 GPU（SM ≥ 80）原生支持；在不支持的显卡上会触发 CUDA 编译错误或运行时崩溃。当前代码未做显卡特性检测，仅依赖用户自行配置。

### 💡 关注建议  

| 对象 | 建议 |
|------|------|
| **开发者** | - 在新增的 `flash_c*_decode / flash_c*_prefill` 调用链中，保持 `BufferFloat` 与 `InputFloat` 参数顺序一致，防止模板实例化错误。<br>- 若计划支持 **fp16** 或 **int8** 状态，建议抽象 `cast` 为统一工具函数，避免在多个 kernel 中重复实现。 |
| **运维/使用者** | - 如需显存节省且模型不使用在线 c128（即 `SGLANG_OPT_USE_ONLINE_COMPRESS=0`），可安全开启 `SGLANG_DSV4_COMPRESS_STATE_DTYPE=bf16`。<br>- 在 GPU 为 Ampere+（支持 BF16）时开启；在旧卡上请保持默认 `float32`，或在部署脚本中加入显卡特性检测。 |
| **测试/CI** | - 将 `test_deepseek_v4_compress_state_runtime_shapes.py` 加入每日 CI，确保在不同 CUDA 版本、V100/A100 等硬件上仍能通过数值误差阈值。<br>- 增加 **压力测试**（如 batch = 1、seq_len = 8192）来捕获极端混合 dtype 路径的潜在性能回退。 |
| **文档** | - 在项目 README / 环境变量说明中明确 `SGLANG_DSV4_COMPRESS_STATE_DTYPE` 的取值、限制（不可在在线压缩开启时使用 bf16），以及对显存、吞吐的预期影响。 |

**结论**：此次提交在 DeepSeek V4 压缩实现上引入了 **mixed‑precision 状态池**，为大模型部署提供显存节省与潜在吞吐提升的可选路径，同时保持向后兼容。风险主要集中在数值精度与特定小 batch 场景的轻度性能回退上，建议通过文档指引、CI 检测以及硬件特性检查来降低风险。整体而言，是一次 **高价值、高风险** 的功能增强，值得在主线合并并配合充分的回归测试。

### [diffusion] fix: use Megatron-style tp for native encoders and dits (#28318)
**SHA**: `0f5e14e` | 🔗 [查看提交](https://github.com/sgl-project/sglang/commit/0f5e14e1d9ff07b9627204450ce9576f42f6cc58)

**🎯 变更类型**：架构变更 / 性能优化 / 重构  
**⚡ 重要程度**：🔴高  

**📋 变更摘要**  
本次提交为 sglang 项目引入了 **Megatron‑style Tensor Parallel (TP)** 支持，覆盖了原生文本/视觉 encoder、DiT 系列模型以及部分 pipeline 配置。关键改动包括：  
1. 新增 `ColumnParallelLinear`、`RowParallelLinear`、`MergedColumnParallelLinear`、`WeightOnlyFP8*` 系列实现，并在权重加载、sharding、梯度聚合等环节加入 TP‑aware逻辑。  
2. 在注意力、MLP、LayerNorm、KV 缓存等模块中加入 head‑slice、head‑start、局部 head 数等参数，实现 **跨 TP 分片的自注意力**。  
3. 在 `device_communicators` 中加入维度可整除性断言，防止非法 TP 划分。  
4. 更新多模态 pipeline（如 hunyuan、hunyuanvideo）以及多个 DiT（WanVideo、Cosmos3、GlmImage、JoyImage、LingBotWorld、Ideogram）模型，使其在 TP 环境下使用 column‑/row‑parallel 线性层并正确切分 QKV、FFN 权重。  
5. 为 Mistral‑3、Qwen2.5VL、Qwen‑3VL 等 encoder 添加 TP‑aware线性层，保持与原模型保持功能等价。  
6. 增加若干辅助函数（`_tp_world_size`、`_ensure_tensor_model_output` 等）以及测试用例的 TP‑路径断言。  

**🎯 影响范围**  
- **核心层**：`runtime/layers/linear.py`、`runtime/layers/quantization/weight_only_fp8.py`  
- **分布式通信**：`runtime/distributed/device_communicators/base_device_communicator.py`  
- **模型实现**：`runtime/models/dits/*`（WanVideo、Cosmos3、GlmImage、JoyImage、LingBotWorld、Ideogram、HunyuanVideo 等）  
- **Encoder**：`runtime/models/encoders/mistral_3.py`、`runtime/models/encoders/qwen2_5vl.py`、`runtime/models/encoders/qwen3vl.py`  
- **Pipeline 配置**：`multimodal_gen/configs/pipeline_configs/hunyuan.py`  
- **测试**：`test/unit/test_ideogram4.py`、`pipelines_core/stages/denoising.py`（返回统一 Tensor）  

**🔍 技术洞察**  

| 维度 | 影响详情 |
|------|----------|
| **架构影响** | - 引入 **列并行（ColumnParallel）** 与 **行并行（RowParallel）** 线性层，模型权重从单卡完整存储转为 **TP‑shard**；<br>- QKV、FFN、投影层的 `gather_output` 与 `input_is_parallel` 标记决定是否在前向/反向阶段进行 all‑gather / all‑reduce。<br>- 多模型（DiT、Encoder）统一使用 **局部 head 数** (`local_num_heads`) 以及 **head_start** 来切分 KV 缓存，兼容已有 SP（序列并行）实现。<br>- 通过 `get_tp_world_size`、`get_tp_rank` 动态获取 TP 参数，保持单卡/多卡代码兼容。 |
| **性能影响** | - **显存**：每张卡仅保存本地 shard，显著降低大模型（如 30B+）的显存占用。<br>- **计算**：线性层的矩阵乘法在更小的维度上完成，理论上可实现 **线性加速**（受通信开销限制）。<br>- **通信**：在 `forward` 中加入 `tensor_model_parallel_all_reduce`/`all_gather`，以及 KV 缓存的 head‑slice 填充，带来额外的 NCCL/通信开销；在高带宽、低 latency 环境下可实现接近理想加速，劣势在跨节点网络不佳时可能出现瓶颈。 |
| **安全考虑** | - 代码仅涉及内部张量切分与通信，无外部输入校验变化；<br>- 新增的断言（如 head‑dim 必须能被 TP 整除）可以在启动阶段提前捕获配置错误，提升安全性。<br>- 未出现新的 I/O、文件或网络暴露路径，安全风险基本为 **0**。 |
| **可维护性** | - 大量新增抽象层（`_make_column_linear`、`_make_row_linear`、`_linear_output` 等），对老代码的侵入度降低，后续可以统一 TP 与非 TP 逻辑。<br>- 代码路径分支增多（`if self.use_tensor_parallel:`），需要在调试时注意 TP 与非 TP 两套路径的行为一致性。 |
