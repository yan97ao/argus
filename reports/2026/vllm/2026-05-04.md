### [CI] Clean up remote servers on pytest parent exit (#41570)
**SHA**: `67058ca` | 🔗 [查看提交](https://github.com/vllm-project/vllm/commit/67058ca326ac566db1aac6fc20ff0c43510422f8)

**🎯 变更类型**：其他（CI / 资源清理）  
**⚡ 重要程度**：🟡 中  

**📋 变更摘要**  
- 在 `tests/utils.py` 中为 `RemoteVLLMServer` 增加全局活动服务器集合、锁、以及一次性注册的 `atexit` 与信号钩子。  
- 实例化时会把服务器登记到集合，销毁时自动注销；父进程在收到 `SIGTERM`/`SIGINT` 或进程退出时会遍历集合并调用 `_shutdown`，确保即使测试中出现异常也能释放子进程和 GPU 内存。  
- `shutdown_many` 现在会提前标记 `_shutdown_complete` 并在完成后统一注销，避免重复关闭。  

**🎯 影响范围**  
- `RemoteVLLMServer` 的生命周期管理（所有调用该类的单元/集成测试）。  
- CI 环境下的 pytest 运行，尤其是并行或长时间运行的测试会受益于更可靠的子进程回收。  
- 与信号处理相关的跨平台行为（Linux/macOS 为主，Windows 仍使用 `atexit`）。  

**💡 关注建议**  
1. **线程安全**：已使用 `RLock` 包装集合操作，保持原子性；若在多进程环境（`pytest-xdist`）中使用，需要确认每个进程拥有独立集合。  
2. **信号兼容性**：`signal` 在 Windows 只支持少数信号，建议在 `if hasattr(signal, "SIGTERM")` 条件下注册，以免在 Windows CI 上抛异常。  
3. **冗余关闭**：`_shutdown_complete` 防止二次调用，若未来增加异步关闭路径，请同步更新该标记。  
4. **测试验证**：加入单元测试，模拟父进程提前退出、手动发送 SIGINT，确认子进程被成功 terminate 并且不会留下僵尸进程。  
5. **文档/使用提示**：在 `README` 或测试指南中说明：在自定义脚本中直接使用 `RemoteVLLMServer` 时，无需额外清理，库会在进程退出或信号到达时自动回收。  

总体来说，此次改动提升了 CI 环境下的资源可靠回收，降低了因残留子进程导致的 GPU OOM 或端口占用风险，风险较低，只需关注跨平台信号兼容性和相应的测试覆盖。

### [Bugfix] Fix degenerate KV cache stride causing TMA cudaErrorIllegalInstruction (#40737)
**SHA**: `66dfee7` | 🔗 [查看提交](https://github.com/vllm-project/vllm/commit/66dfee7121dfcbdfcce04ce92c117e5e10b25b14)

**🎯 变更类型**：Bug 修复  
**⚡ 重要程度**：🟡 中  

**📋 变更摘要**  
- 新增 `canonicalize_singleton_dim_strides()`，在张量存在 size‑1 维度时，将其 stride 修正为 “后续维度乘积”，满足 H100+ CUDA TMA 对 ≥16 byte 对齐的要求。  
- 在 FlashAttention、FlashAttention‑DiffKV、FlashInfer 三大注意力后端以及相应的 prefill / decode 路径中调用该函数并加入调试日志。  
- 添加完整单元测试覆盖各种布局、不同块尺寸、多个 singleton 维度等场景。  
- 小幅改动 `cpu_resource_utils` 以兼容 macOS 上的 `os.sched_getaffinity` 类型检查。  

**🎯 影响范围**  
- `vllm/utils/torch_utils.py`（核心实现）  
- `vllm/v1/attention/backends/flash_attn.py`、`flash_attn_diffkv.py`、`flashinfer.py`（KV‑cache 读取路径）  
- 新增测试 `tests/v1/attention/test_kv_head_stride_canonicalization.py`  
- 轻微影响 `vllm/utils/cpu_resource_utils.py`  

**💡 关注建议**  
1. **代码审查**：确认 `as_strided` 生成的视图在所有后端（包括自定义 CUDA kernel）中可安全使用；目前已通过单元测试，但在实际推理过程中请关注可能的内存布局限制。  
2. **性能验证**：该函数仅修改元数据，理论上无额外拷贝开销，但建议在大模型/高并发场景跑一次基准，确保没有意外的 GPU‐kernel 重编译或 TMA descriptor 重新创建。  
3. **向后兼容**：对没有 size‑1 维度的张量保持原对象返回，避免不必要的视图创建；若未来新增自定义 attention 实现，请记得在对应 KV‑cache 读取后调用 `canonicalize_singleton_dim_strides`。  
4. **测试维护**：新加入的测试覆盖了 bf16、fp16、不同 block/head 大小以及多 singleton 维度，后续若修改 KV‑cache 布局，请相应更新或补充对应的测试用例。  

总体而言，此次修改针对 H100+ TMA 需求的 stride 对齐 bug，修复可靠且对现有功能影响有限，建议合并后在真实硬件上做一次完整的推理/生成跑通验证。
