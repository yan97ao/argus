### docs: restructure docs directory and move fern config to fern/ (#6700)
**SHA**: `ece08dc` | 🔗 [查看提交](https://github.com/ai-dynamo/dynamo/commit/ece08dc926679909dd7a0d980f073cdee47414c2)

**📝 变更概览**  
本次 PR 彻底重构了 Dynamo 的文档体系：

1. **目录结构**  
   - 所有 Markdown 从 `docs/pages/` 移至 `docs/`，并把原来的 `dev.yml` 替换为 `index.yml`。  
   - Fern‑CLI 的配置文件统一放在 `fern/`（`fern.config.json`、`docs.yml`、`components/`、`main.css` 等），不再依赖 `docs/` 下的 `fern` 符号链接。  

2. **CI / 同步流程**  
   - `.github/workflows/fern-docs.yml` 改为在 `repo root` 执行 `fern check`、`fern docs broken-links`，并用 `rsync`/`yq` 将 `docs/` 内容同步到 `docs‑website` 分支的 `fern/pages‑dev/` 与 `fern/versions/dev.yml`，同时完成路径前缀转换（`getting‑started/quickstart.md → ../pages‑dev/getting‑started/quickstart.md`）。  
   - 版本发布流程也相应改为操作 `fern/pages‑vX.Y.Z/` 与 `fern/versions/vX.Y.Z.yml`。  

3. **文档链接全量迁移**  
   - 近千处相对链接、图片路径、`docs.yml`、`README.md` 等全部改写，从 `docs/pages/...` → `docs/...`，从 `docs/versions/dev.yml` → `docs/index.yml`，从 `docs/fern/...` → `fern/...`。  
   - 相关代码、示例、教程、博客等全部同步更新。  

4. **其它细节**  
   - `.github/filters.yaml` 中的路径过滤同步改动。  
   - 删除旧的 `docs/versions/dev.yml`，新增 `docs/index.yml`（357 行完整导航）。  
   - 多个 Rust/Python 代码中对文档位置的注释也相应调整。  

**⚡ 影响范围**  
- **文档生成与发布**：所有依赖 `fern check`、`fern docs broken-links` 的本地和 CI 验证需要在根目录运行；若仍使用 `cd docs && ln -s . fern` 的旧方式会失效。  
- **CI/CD**：`fern-docs.yml` 关键步骤（路径复制、`yq` 转换、`rsync`）已改动，任何自定义步骤仍假设旧结构的会报错。  
- **README / 示例**：对外文档、快速入门、示例脚本等的链接已更新，未同步的仓库镜像或外部文档（如博客、第三方教程）仍指向旧路径，可能产生 404。  
- **代码注释**：仅是说明文字，不影响运行，但若 CI 中有自动检查文档引用的脚本，需同步更新路径规则。  

**🔧 建议**  
1. **本地验证**：在仓库根目录执行 `fern check`、`fern docs broken-links`、`fern docs dev`，确认无路径错误后再提交。  
2. **CI 预跑**：在 PR 中打开 `Fern Docs` 工作流的 “Run workflow” 手动触发，检查 `sync‑dev` 与 `release‑version` 两个 Job 的日志，尤其是 `yq` 路径替换是否如预期。  
3. **链接完整性**：利用 `detect_broken_links.py` 或 `fern docs broken-links` 再次全局扫描，确保所有 `../blogs/`、`../assets/`、`../diagrams/` 等引用已更新。  
4. **文档外部引用**：若有外部站点（如 DeepWiki、社区论坛）硬编码了旧路径，请通知维护者或在仓库根目录添加短链（GitHub Pages 重定向）以避免用户 404。  
5. **脚本兼容**：凡是使用 `cd docs && ...` 的自定义脚本（例如本地调试或 CI 步骤）需要改为根目录执行，或者在脚本内部先 `pushd docs && ln -s . fern && popd` 以保持向后兼容。  

**结论**  
此次重构提升了文档与 Fern 配置的统一性，消除原先的 “docs/fern → docs/. ” 符号链接 hack，后期维护更简洁。但它对 CI、示例脚本以及任何硬编码文档路径的地方都有连锁影响。务必在本地完整跑一遍 `fern` 检查、同步工作流以及示例部署脚本，确保所有路径已成功迁移，避免发布后出现文档失联或 CI 失效的情况。
