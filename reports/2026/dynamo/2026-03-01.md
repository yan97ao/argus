### ⚠️ 潜在风险

1. **误用自动化脚本**  
   - `pr-monitor`、`gh-issue-bug`、`debug-session` 均给出 **CLI 命令**（如 `gh pr view`、`curl`），如果用户在不受信任的终端直接复制粘贴，可能会泄露敏感 token（如 `gh auth token`）或触发意外网络请求。  
   - **建议**：在 Skill 文档中明确提醒「**仅在受信任环境**」执行，或在模板中使用占位符 `{{TOKEN}}` 让用户自行填充。

2. **.gitignore 误配置**  
   - 目前规则是 `/.claude/*` + `!/.claude/skills/`，意味着 **所有** `.claude` 目录下的文件都被忽略，只有 `skills/` 中的文件会被提交。若未来在 `.claude` 里添加临时缓存或日志，可能不小心被提交或被误删。  
   - **建议**：在团队文档中说明 `.claude/` 为 **“仅本地存放 Claude Agent 数据”**，如需新增子目录请同步更新 `.gitignore`。

3. **文档同步**  
   - 这些 Skill 与实际代码（如 `lib/parsers/...`）保持强关联。如果未来工具链或 CI 流程变更，Skill 文档可能会过时。  
   - **建议**：在每次对应代码模块重大改动（如 parser 目录结构）时，使用 CI 检查文档是否仍然匹配（例如通过 `markdownlint` +自定义脚本比较关键字段）。

4. **依赖外部 CLI**  
   - Skill 中大量使用 `gh`、`curl`、`nvidia-smi`、`uname` 等外部工具。若在 CI 环境运行（误触发）可能导致 **未安装** 或 **权限错误**。  
   - **建议**：在 Skill 顶部加入 “**依赖前置**” 小节，列出必备 CLI 并提供安装指令。

**🎯 变更类型**：性能优化 / 重构 / 架构变更  

**⚡ 重要程度**：🔴高  

**📋 变更摘要**  
1. 将 `ConcurrentRadixTree` 的全局 per‑worker lookup (`RwLock<FxHashMap<WorkerWithDpRank,…>>`) 替换为 **线程本地**的 `FxHashMap`，并通过 `dashmap::DashMap` 保存每个 worker 的块数量（`tree_sizes`），大幅降低并发写入时的锁竞争。  
2. 为所有 `SyncIndexer` 实现新增 `worker(event_receiver)` 接口，统一使用 **任务枚举 `WorkerTask`**（Event / RemoveWorker / DumpEvents / Terminate）在后台线程上处理，`ThreadPoolIndexer` 相应改为基于该枚举进行消息调度。  
3. `PositionalIndexer`（旧的嵌套 map）同步改写，同样搬迁 per‑worker 反向索引到 **线程本地**，并使用 `tree_sizes` 维护块计数。  
4. 移除 `metrics` 可选特性、删除冗余基准代码、在 Dockerfile 中加入 `libfontconfig-dev`，进一步缩减二进制体积与构建时间。  

**🎯 影响范围**  
- `lib/kv-router/src/concurrent_radix_tree.rs`（核心并发路由树）  
- `lib/kv-router/src/nested_map.rs`（嵌套 map 实现）  
- `lib/kv-router/src/indexer.rs`（`SyncIndexer` 与 `ThreadPoolIndexer`）  
- `lib/kv-router/Cargo.toml`（特性与依赖）  
- 删除的基准文件 `radix_tree_microbench.rs`  
- `container/templates/dev.Dockerfile`（新增系统库）
